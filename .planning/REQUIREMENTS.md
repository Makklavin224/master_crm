# Requirements: Мастер-CRM

**Defined:** 2026-03-17
**Core Value:** Мастер нажимает "Завершить" — клиент получает ссылку на оплату — чек формируется автоматически. 3 шага вместо 9.

## v1 Requirements

### Booking (BOOK)

- [ ] **BOOK-01**: Мастер может создать каталог услуг (название, длительность, цена, категория)
- [ ] **BOOK-02**: Мастер может настроить расписание (рабочие часы по дням, перерывы, выходные)
- [ ] **BOOK-03**: Клиент может видеть свободные слоты и записаться (выбор услуги → даты → времени)
- [ ] **BOOK-04**: Система предотвращает двойную запись (PostgreSQL exclusion constraints)
- [ ] **BOOK-05**: Клиент может отменить или перенести запись (с настраиваемым дедлайном)
- [ ] **BOOK-06**: Мастер может отменить, перенести или вручную добавить запись
- [ ] **BOOK-07**: Мастер получает уведомление о новой записи/отмене через бот

### Clients (CLNT)

- [ ] **CLNT-01**: Клиентская база заполняется автоматически из записей (имя, телефон)
- [ ] **CLNT-02**: Клиент идентифицируется по номеру телефона (E.164 нормализация)
- [ ] **CLNT-03**: Мастер видит историю визитов клиента
- [ ] **CLNT-04**: Клиент из разных мессенджеров распознаётся как один человек по телефону

### Notifications (NOTF)

- [ ] **NOTF-01**: Автоматическое напоминание клиенту за 24ч до визита через бот
- [ ] **NOTF-02**: Автоматическое напоминание клиенту за 2ч до визита через бот
- [ ] **NOTF-03**: Подтверждение записи клиенту сразу после бронирования

### Payments (PAY)

- [ ] **PAY-01**: Мастер может завершить визит и отметить "Оплачено" (простой режим, оплата на карту)
- [ ] **PAY-02**: Мастер может опционально подключить свою Robokassa для автоматической ссылки СБП
- [ ] **PAY-03**: При подключённой Robokassa: мастер нажимает "Завершить" → клиент получает ссылку на оплату
- [ ] **PAY-04**: История платежей со статусами (ожидает, оплачено, отменено)

### Tax Receipts (TAX)

- [ ] **TAX-01**: Три уровня фискализации на выбор мастера: без чеков / ручной / автоматический
- [ ] **TAX-02**: Без чеков: мастер видит предупреждение что лучше выставлять чеки, но может продолжить
- [ ] **TAX-03**: Ручной: после оплаты CRM показывает готовые данные для "Мой Налог" (сумма, услуга, клиент)
- [ ] **TAX-04**: Автоматический: при подключённой Robokassa чек формируется через Robochecks СМЗ → ФНС

### Messengers (MSG)

- [ ] **MSG-01**: Telegram Bot — обработка webhook, уведомления мастеру и клиентам
- [ ] **MSG-02**: Telegram Mini App — запись клиента, управление мастером
- [ ] **MSG-03**: MAX Bot — обработка webhook, уведомления
- [ ] **MSG-04**: MAX Mini App — та же функциональность что TG Mini App
- [ ] **MSG-05**: VK Bot — обработка webhook, уведомления
- [ ] **MSG-06**: VK Mini App — та же функциональность что TG Mini App
- [ ] **MSG-07**: Единый React Mini App код с platform detection (TG/MAX/VK bridge adapters)
- [ ] **MSG-08**: Messenger Adapter pattern на бэкенде (единый notification router)

### Web Panel (WEB)

- [ ] **WEB-01**: Веб-панель мастера — календарь и список записей (день/неделя)
- [ ] **WEB-02**: Веб-панель — управление клиентами (список, история визитов)
- [ ] **WEB-03**: Веб-панель — CRUD услуг (создание, редактирование, удаление, категории)
- [ ] **WEB-04**: Веб-панель — история платежей и статусы чеков
- [ ] **WEB-05**: Веб-панель — настройки расписания и профиля мастера

### Infrastructure (INFR)

- [ ] **INFR-01**: FastAPI backend с PostgreSQL, развёрнутый в Docker Compose на VPS
- [ ] **INFR-02**: REST API для mini-app и web panel
- [ ] **INFR-03**: Webhook-обработчики для TG, MAX, VK ботов
- [ ] **INFR-04**: Мастер-аутентификация (регистрация/логин через мессенджер)
- [ ] **INFR-05**: Multi-tenant изоляция (master_id на всех таблицах, PostgreSQL RLS)

## v2 Requirements

### Analytics

- **ANLYT-01**: Доход за период (день/неделя/месяц)
- **ANLYT-02**: Количество записей и конверсия
- **ANLYT-03**: Процент повторных клиентов

### Client Engagement

- **ENGMT-01**: Заметки и теги по клиентам
- **ENGMT-02**: Приглашения на повторный визит ("Вы не были 30 дней")
- **ENGMT-03**: Фото-портфолио мастера в мини-аппе

### Finance

- **FIN-01**: Учёт доходов/расходов
- **FIN-02**: Экспорт отчётов (PDF/Excel)

### Growth

- **GRWTH-01**: Подписочная модель (Free/Pro/Business)
- **GRWTH-02**: Онбординг-визард для нового мастера
- **GRWTH-03**: Сбор и отображение отзывов

## Out of Scope

| Feature | Reason |
|---------|--------|
| Склад/остатки | Оверкилл для solo-мастера |
| Расчёт зарплат | CRM для одиночек, не для салонов |
| Программы лояльности | Сложно, низкий ROI для solo |
| Телефония | Мессенджеры заменяют |
| Управление филиалами | Solo-фокус |
| Нативное мобильное приложение | Мини-аппы в мессенджерах заменяют |
| Marketplace/каталог мастеров | Другой бизнес, требует двустороннего рынка |
| Email-рассылки | Клиенты в мессенджерах |
| AI-чатбот для клиентов | Преждевременная сложность, бот-меню быстрее |
| Сложная аналитика (воронки) | Solo не аналитики, "сколько заработал" достаточно |
