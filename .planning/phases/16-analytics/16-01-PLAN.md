    Create `backend/app/schemas/analytics.py` with Pydantic models:
    - `AnalyticsSummary`: revenue (int, kopecks), booking_count (int), client_count (int), new_clients (int), repeat_clients (int), utilization (float, 0-100%), avg_check (int, kopecks), retention_rate (float, 0-100%), cancel_rate (float, 0-100%), noshow_rate (float, 0-100%)
    - `RevenueChartPoint`: date (str, "YYYY-MM-DD"), revenue (int, kopecks)
    - `TopServiceRow`: service_name (str), booking_count (int), revenue (int, kopecks), percentage (float, 0-100)
    - `DailyBreakdownRow`: date (str, "YYYY-MM-DD"), booking_count (int), revenue (int, kopecks), utilization (float, 0-100%)

    Create `backend/app/services/analytics_service.py` with class `AnalyticsService` using @staticmethod methods (same pattern as PaymentService):

    1. `get_summary(db, master_id, date_from, date_to)` -> AnalyticsSummary:
       - Revenue: SUM of Payment.amount WHERE status='paid' AND paid_at in range
       - Booking count: COUNT of Booking WHERE starts_at in range (all statuses except cancelled)
       - Client count: COUNT DISTINCT client_id from Booking WHERE starts_at in range
       - New vs repeat: Query master_clients WHERE first_visit_at in range (new), vs first_visit_at < range start AND last_visit_at in range (repeat). Use bookings in range joined to master_clients.
       - Utilization: (SUM of Booking duration_minutes for completed bookings in range) / (scheduled working hours in range * 60). Scheduled hours = iterate each date in range, find MasterSchedule for that day_of_week (is_working=True), subtract breaks, check ScheduleException overrides. If no schedule data, return 0.
       - Avg check: revenue / count of paid payments in range. If 0 payments, return 0.
       - Retention: Among clients who had a completed booking before date_from, what % had another completed booking in [date_from, date_to]. Use master_clients: clients with first_visit_at < date_from AND visit_count > 0, then check if they have a booking in range. Simpler: count clients with completed bookings in range whose first_visit_at with this master < date_from, divide by total clients who had visits before date_from. If no prior clients, return 0.
       - Cancel rate: COUNT bookings with status in ('cancelled_by_client', 'cancelled_by_master') / total bookings in range * 100
       - No-show rate: COUNT bookings with status='no_show' / total bookings in range * 100

    2. `get_revenue_chart(db, master_id, date_from, date_to)` -> list[RevenueChartPoint]:
       - GROUP BY date(paid_at), SUM(amount) WHERE status='paid' AND paid_at in range
       - Fill gaps: for each date in range that has no payments, include {date, revenue: 0}

    3. `get_top_services(db, master_id, date_from, date_to)` -> list[TopServiceRow]:
       - JOIN bookings to services and payments (status='paid')
       - GROUP BY service.name, COUNT bookings, SUM payment amount
       - Calculate percentage = service_revenue / total_revenue * 100
       - ORDER BY revenue DESC

    4. `get_daily_breakdown(db, master_id, date_from, date_to)` -> list[DailyBreakdownRow]:
       - GROUP BY date(starts_at) from bookings in range (exclude cancelled)
       - For each date: count bookings, sum paid payment amounts, calculate utilization (completed duration / scheduled hours for that specific day)
       - ORDER BY date ASC

    All queries use `select`, `func`, `and_` from sqlalchemy. Import models directly inside methods (same pattern as payment_service.py line 539). Use `await db.execute()` for all queries.

    For date range filtering use `func.date(Payment.paid_at)` or `func.date(Booking.starts_at)` with `>=` and `<=` operators (same pattern as payment_service line 547-549).
