    1. **Update frontend/src/stores/master-auth.ts**:
       - Add `role: "master" | "client" | "detecting"` to MasterAuthState (default: "detecting").
       - Add `setRole: (role: "master" | "client") => void` action.
       - Add `autoDetectRole: (bridge: PlatformBridge) => Promise<void>` action that:
         a. Sets role to "detecting", isLoading to true.
         b. First, calls `hydrate()` -- if token exists in localStorage, set role="master", isLoading=false, return.
         c. Gets initDataRaw from bridge.getInitDataRaw(). If null (web platform or no init data), set role="client", isLoading=false, return.
         d. Determines the auth endpoint based on bridge.platform: "telegram" -> "/auth/tg" with `{init_data: initDataRaw}`, "max" -> "/auth/max" with `{init_data: initDataRaw}`, "vk" -> "/auth/vk" with `{launch_params: initDataRaw}`.
         e. Calls the appropriate endpoint. If returns 200 with access_token: store token, set isAuthenticated=true, role="master".
         f. If returns 401/404: set role="client", isAuthenticated=false. This is NORMAL -- means user is not a master.
         g. Sets isLoading=false in all cases.
       - Existing `login(initDataRaw)` function stays as-is (it's TG-specific and used elsewhere).

    2. **Create frontend/src/components/RoleSwitcher.tsx**:
       - Import useMasterAuth store, useNavigate from react-router-dom.
       - Renders a floating pill button fixed at top-right (below safe area):
         - When role="master" and current path starts with "/master": show "👤 Мои записи" button. Clicking sets role="client" and navigates to "/my-bookings".
         - When role="master" and current path does NOT start with "/master": show "⚙️ Панель мастера" button. Clicking sets role="master" and navigates to "/master/dashboard".
         - When role="client": do not render (return null). Clients cannot access master panel.
       - Styling: fixed position, top 12px right 12px, z-50, bg-white rounded-full shadow-md px-3 py-1.5 text-sm font-medium border border-border. Use Tailwind classes matching existing design tokens.
       - Use `useLocation()` to determine current path for showing correct button label.

    3. **Update frontend/src/App.tsx**:
       - Replace the current `EntryRedirect` component with a `RoleDetector` component:
         a. `RoleDetector` uses `usePlatform()` to get the bridge and `useMasterAuth()` for state.
         b. On mount (useEffect), calls `useMasterAuth.getState().autoDetectRole(bridge)`.
         c. While `role === "detecting"`: shows a centered spinner (div with "animate-spin" and a simple loading indicator).
         d. When `role === "master"` and no `?master=` param: navigates to `/master/dashboard`.
         e. When `role === "client"`: checks `?master=` param. If present, navigates to `/book/{masterId}`. If absent, navigates to `/my-bookings`.
         f. When `role === "master"` and `?master=` param IS present: still navigate to `/book/{masterId}` (master is acting as client for another master).
       - Remove the old `useEffect(() => { useMasterAuth.getState().hydrate() })` from App component (hydrate is now called inside autoDetectRole).
       - Add `<RoleSwitcher />` inside BrowserRouter, after Routes (so it renders on all pages).
       - Keep ALL existing routes unchanged.

    4. **Update frontend/src/lib/constants.ts**:
       - No changes needed (routes are already defined).

    IMPORTANT: The auto-detection must be FAST. The auth endpoint call is a single POST. If it fails with 401/404, that's the "not a master" signal -- do NOT show an error toast or log it as an error. It's expected behavior for clients.

    IMPORTANT: Do NOT remove or modify the existing `login(initDataRaw)` function in master-auth.ts. Other code may depend on it. The new `autoDetectRole` is additive.
