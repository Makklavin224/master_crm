Backend platform linking API + enhanced bot handlers for master recognition and registration.

    1. Add to backend/app/schemas/settings.py:
       - `PlatformStatus` Pydantic model: `tg_linked: bool`, `max_linked: bool`, `vk_linked: bool`, `tg_user_id: str | None`, `max_user_id: str | None`, `vk_user_id: str | None`
       - `PlatformUnlinkResponse` model: `ok: bool`, `platform: str`

    2. Add to backend/app/schemas/auth.py:
       - `BotRegisterRequest` model: `name: str`, `email: str`, `platform: str` (literal "telegram"|"max"|"vk"), `platform_user_id: str`
       - `LinkAccountRequest` model: `email: str`, `platform: str` (literal "telegram"|"max"|"vk"), `platform_user_id: str`

    3. Add to backend/app/api/v1/settings.py three new endpoints:
       - `GET /settings/platforms` (auth required via get_current_master): returns PlatformStatus built from master.tg_user_id, master.max_user_id, master.vk_user_id (linked = not None)
       - `DELETE /settings/platforms/{platform}` (auth required): where platform is "telegram"|"max"|"vk". Sets the corresponding user_id column to None on the master. Returns PlatformUnlinkResponse. Validate platform is one of the three, return 400 otherwise.
       - `PUT /settings/platforms/{platform}` is NOT needed here (linking happens through the bot, not web admin directly, since we need the platform_user_id which only the bot has)

    4. Add to backend/app/api/v1/auth.py two new endpoints:
       - `POST /auth/bot-register`: accepts BotRegisterRequest. Validates email format, checks email not already taken (409 if taken). Creates Master with name, email (lowercase), and sets the platform_user_id column based on platform field (e.g., if platform="telegram", set tg_user_id=platform_user_id). No password required (bot-registered masters can set password later via web admin or use bot auth). Returns TokenResponse with JWT.
       - `POST /auth/link-account`: accepts LinkAccountRequest. Looks up Master by email (case-insensitive). If not found, return 404 "Account not found". If found, check the platform_user_id column is not already set to a DIFFERENT value (if already set to same value, that's fine). If column is null, set it to the provided platform_user_id. Return TokenResponse with JWT. If the platform_user_id is already used by ANOTHER master (unique constraint), return 409 "Platform account already linked to another master".

    5. In auth_service.py, add helper `register_master_from_bot(db, name, email, platform, platform_user_id) -> tuple[Master, str]` that encapsulates the creation logic without requiring a password.
  </action>
