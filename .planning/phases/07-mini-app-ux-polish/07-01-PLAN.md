<task type="auto">

  <action>
In frontend/src/index.css, add Telegram theme variable mapping AFTER the existing `html, body, #root` block but BEFORE the `body { overscroll-behavior }` block.

Add this CSS block:

```css
/* Telegram Theme Integration */
:root {
  /* Map Telegram theme variables to design tokens */
  --app-bg: var(--tg-theme-bg-color, #FFFFFF);
  --app-text: var(--tg-theme-text-color, #1A1A2E);
  --app-hint: var(--tg-theme-hint-color, #6B7280);
  --app-link: var(--tg-theme-link-color, var(--color-accent));
  --app-button: var(--tg-theme-button-color, var(--color-accent));
  --app-button-text: var(--tg-theme-button-text-color, #FFFFFF);
  --app-secondary-bg: var(--tg-theme-secondary-bg-color, #F4F5F7);
}

html, body, #root {
  background-color: var(--app-bg);
  color: var(--app-text);
}
```

IMPORTANT: The second `html, body, #root` block with the `var(--app-bg)` MUST come AFTER the first static one (which sets the font-family and smoothing). CSS cascade means the second block's background-color and color will override the first block's values. Keep all properties from the first block (height, margin, padding, font-family, smoothing). The second block ONLY overrides background-color and color with the TG-aware variables.

Also update the existing `--color-surface` in the @theme block:
- Keep `--color-surface: #F4F5F7;` as-is (it remains the fallback)

Also update the `--color-accent-light` to use the new accent:
- Change `--color-accent-light: #6C5CE714;` to `--color-accent-light: #5A4BD114;`
