  **2. web/src/components/CompleteVisitModal.tsx — new file:**

  Props: `{ open: boolean; booking: BookingRead | null; onClose: () => void; onSuccess: () => void }`.

  Component logic:
  - Uses `useServices()` to look up service price by `booking.service_id`. Pre-fill amount as `servicePrice / 100` (display in rubles, store in kopecks).
  - Uses `usePaymentSettings()` to determine: (a) whether to show SBP option (only if `has_robokassa`), (b) whether to show fiscalization section (only if `has_robokassa`), (c) default fiscalization from settings.
  - State: `amount` (number, rubles for display), `paymentMethod` ("cash" | "card_to_card" | "sbp"), `fiscalization` ("auto" | "manual" | "none").
  - Default paymentMethod: "cash".
  - Default fiscalization: from settings `fiscalization_level`, or "none" if not set.

  UI (Ant Design Modal):
  - Title: "Завершить визит"
  - **Сумма** section: `InputNumber` with addonAfter="₽", min=0, precision=0, value=amount. Pre-filled from service price / 100.
  - **Способ оплаты** section: `Radio.Group` with `optionType="button"` (pill style), `buttonStyle="solid"`. Options: "Наличные" (cash), "Перевод на карту" (card_to_card). If `paymentSettings.has_robokassa`: also show "СБП" (sbp).
  - **Чек** section (shown ONLY if `paymentSettings.has_robokassa`): `Radio.Group` with `optionType="button"`, `buttonStyle="solid"`. Options: "Авто" (auto), "Вручную" (manual), "Без чека" (none). If "none" selected, show orange Alert below: "Вы выбрали «Без чека». Чек не будет отправлен в ФНС."
  - OK button text: "Завершить и принять оплату", okButtonProps color green (#00B894).
  - Cancel button text: "Отмена".

  Submit logic:
  - Convert amount from rubles to kopecks: `amountKopecks = Math.round(amount * 100)`.
  - If paymentMethod is "sbp" AND paymentSettings.has_robokassa: call `useCreateRobokassaPayment` with `{ booking_id, fiscalization_level, amount_override: amountKopecks !== servicePrice ? amountKopecks : undefined }`.
  - Otherwise: call `useCreateManualPayment` with `{ booking_id, payment_method: paymentMethod, fiscalization_level, amount_override: amountKopecks !== servicePrice ? amountKopecks : undefined }`.
  - On success: show `messageApi.success("Визит завершён, оплата принята")`, call `onSuccess()`.
  - On error: show `messageApi.error("Не удалось завершить визит")`.

  NOTE: Both `/payments/manual` and `/payments/robokassa` backend endpoints already set `booking.status = "completed"` internally. Do NOT call the separate `/bookings/{id}/complete` endpoint — it would fail because the booking is already completed.
