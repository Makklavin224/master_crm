Modify `get_current_client` in dependencies.py to support dual-path authentication:

1. First try to get token from cookie: `request.cookies.get("client_session")`
2. If no cookie, try Authorization header: parse `request.headers.get("authorization")`, check it starts with "Bearer ", extract the token part.
3. If neither source has a token, raise 401.
4. The rest of the function (querying ClientSession, checking is_verified + expires_at) stays exactly the same -- it works with the token string regardless of source.

Implementation:
```python
async def get_current_client(
    request: Request,
    db: Annotated[AsyncSession, Depends(get_db)],
) -> Client:
    """Extract client session from cookie OR Bearer token, validate, return Client."""
    # Try cookie first
    token = request.cookies.get("client_session")

    # Fallback to Bearer header
    if not token:
        auth_header = request.headers.get("authorization", "")
        if auth_header.lower().startswith("bearer "):
            token = auth_header[7:].strip()

    if not token:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Not authenticated",
        )
    # ... rest of function unchanged (query ClientSession by token)
```
