    1. **Create payments router** (backend/app/api/v1/payments.py):
       - `POST /manual` -- create manual payment. Accepts PaymentCreate body. Calls PaymentService.create_manual_payment(). Returns PaymentRead. If effective fiscalization is "manual", include receipt_data in response.
       - `POST /robokassa` -- create Robokassa payment. Accepts RobokassaPaymentCreate body. Calls PaymentService.create_robokassa_payment(). Send payment link to client via notification_service.send_payment_link() (fire-and-forget try/except). Returns PaymentRead with payment_url.
       - `POST /requisites` -- create requisites payment. Calls PaymentService.create_requisites_payment(). Returns PaymentRead + PaymentRequisites (card, phone, QR).
       - `POST /{payment_id}/confirm` -- confirm requisites payment. Accepts payment_method. Calls PaymentService.confirm_requisites_payment(). Returns PaymentRead.
       - `POST /{payment_id}/cancel` -- cancel payment. Calls PaymentService.cancel_payment(). Returns PaymentRead.
       - `GET /history` -- list payments. Accepts query params: status, date_from, date_to, limit, offset. Calls PaymentService.get_payment_history(). Returns PaymentListResponse.
       - `GET /{payment_id}/receipt-data` -- get manual receipt data. Returns ManualReceiptData.
       All endpoints require get_current_master + get_db_with_rls dependencies (same pattern as bookings.py).

    2. **Extend settings router** (backend/app/api/v1/settings.py):
       - `GET /payment` -- returns PaymentSettings (card_number, sbp_phone, bank_name, has_robokassa, fiscalization_level, etc.)
       - `PUT /payment` -- accepts PaymentSettingsUpdate, updates master columns, returns PaymentSettings
       - `POST /payment/robokassa` -- accepts RobokassaSetup, encrypts password1/password2 with encryption_service.encrypt_value(), saves to master columns, returns PaymentSettings
       - `DELETE /payment/robokassa` -- clears all robokassa_* columns on master, returns PaymentSettings
       - `POST /payment/grey-warning-seen` -- sets master.has_seen_grey_warning = True, returns 204
       Note: PaymentSettings.has_robokassa is computed as `master.robokassa_merchant_login is not None`.

    3. **Register payments router** (backend/app/api/v1/router.py): Add `from app.api.v1.payments import router as payments_router` and `api_v1_router.include_router(payments_router, prefix="/payments", tags=["payments"])`.

    4. **Add Robokassa webhook** (backend/app/main.py): Add a new route OUTSIDE the api_v1_router (at app level, like the Telegram webhook):
       ```python
       @application.post("/webhook/robokassa/result")
       async def robokassa_result_callback(request: Request):
       ```
       - Parse form data: OutSum, InvId, SignatureValue, Shp_* params
       - Extract Shp_master_id, validate presence
       - Get a DB session (use get_async_session or similar pattern from existing code)
       - Call PaymentService.handle_robokassa_callback()
       - If success: return PlainTextResponse(f"OK{inv_id}")
       - If failure: return PlainTextResponse("signature verification failed", status_code=403)
       This is a PUBLIC endpoint (no auth). Security is via HMAC signature.
