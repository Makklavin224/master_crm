  <action>
    1. Install APScheduler: run `cd backend && uv add "apscheduler>=3.11.0,<4.0.0"` (do NOT use 4.x alpha).

    2. Create `backend/app/services/reminder_service.py`:
       - Import APScheduler AsyncIOScheduler, SQLAlchemy select/and_/insert, async_session_factory, models.
       - Create module-level `scheduler = AsyncIOScheduler(timezone="UTC")`.
       - Implement `async def process_pending_reminders()`:
         a. Open async_session_factory() session.
         b. Get current UTC time: `now = datetime.now(timezone.utc)`.
         c. Query all masters with reminders_enabled=True (SELECT id, name, timezone, reminder_1_hours, reminder_2_hours, address_note, tg_user_id FROM masters WHERE reminders_enabled = true).
         d. For each master, query confirmed bookings where starts_at is in the future AND starts_at <= now + max(reminder_1_hours, reminder_2_hours) hours (upper bound to limit query scope).
         e. For each booking:
            - Check reminder_1: if `booking.starts_at - timedelta(hours=master.reminder_1_hours) <= now` AND starts_at > now:
              - Check if ScheduledReminder exists for (booking_id, "reminder_1") with status in ("sent", "pending"). If yes, skip.
              - Load client's platform_user_id for booking.source_platform via ClientPlatform.
              - If no platform record, skip.
              - Convert starts_at to master's local timezone for display: `local_start = booking.starts_at.astimezone(ZoneInfo(master.timezone))`.
              - Call notification_service.send_reminder() with formatted date/time.
              - On success: INSERT ScheduledReminder(booking_id, master_id, reminder_type="reminder_1", scheduled_for=booking.starts_at - timedelta(hours=master.reminder_1_hours), sent_at=now, status="sent").
              - On failure: INSERT with status="failed", error_message.
            - Check reminder_2 (same logic with reminder_2_hours, reminder_type="reminder_2"). Skip if master.reminder_2_hours is None.
         f. Commit session.
         g. Wrap entire function in try/except, log errors, never crash the scheduler.

       - `scheduler.add_job(process_pending_reminders, "interval", minutes=5, id="reminder_poll", replace_existing=True, misfire_grace_time=300)`.

       - Implement `async def cleanup_reminders_for_booking(db: AsyncSession, booking_id: uuid.UUID)`:
         - DELETE FROM scheduled_reminders WHERE booking_id = :booking_id AND status = "pending".
         - Called from booking_service when booking is cancelled or rescheduled.

    3. Update `backend/app/services/booking_service.py`:
       - In cancel_booking(), after status change and before _notify_master, call: `await cleanup_reminders_for_booking(db, booking_id)` (import from reminder_service). Fire-and-forget pattern (try/except).
       - In reschedule_booking(), after time update and before _notify_master, call: `await cleanup_reminders_for_booking(db, booking_id)`.

    4. Update `backend/app/main.py` lifespan:
       - Import scheduler from reminder_service: `from app.services.reminder_service import scheduler`.
       - After webhook registration (before yield), add:
         ```python
         scheduler.start()
         logger.info("Reminder scheduler started")
         ```
       - In shutdown (after yield), before bot cleanup:
         ```python

         logger.info("Reminder scheduler stopped")
         ```

       - Create master (reminders_enabled=True, reminder_1_hours=24, reminder_2_hours=2), service, client with ClientPlatform(telegram), booking with starts_at = now + 23 hours.
       - Mock `app.bots.common.notification.notification_service.send_reminder` with AsyncMock returning True.
       - Mock `app.services.reminder_service.async_session_factory` to return the test db_session.
       - Call process_pending_reminders() directly.
       - Assert send_reminder was called once with reminder_type containing "1" or appropriate value.
       - Test idempotency: insert ScheduledReminder row, call again, assert send_reminder not called.
       - Test cancelled booking: set booking.status = "cancelled_by_client", call, assert not called.
       - Test past due: set starts_at to 1 hour ago, call, assert not called.
       - Test disabled: set master.reminders_enabled = False, call, assert not called.
       - Test cleanup: call cleanup_reminders_for_booking, verify pending rows deleted.
