# Phase 2 — UI Design Contract

> Visual and interaction contract for the Telegram Mini App (client booking + master management). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Custom components with Tailwind CSS 4.2 utility classes (no shadcn/Radix -- mini-app bundle size constraint) |
| Icon library | Lucide React (`lucide-react`) -- tree-shakeable, 1KB per icon, consistent 24px grid |
| Font | Inter (Google Fonts, `wght@400;600` variable axis subset, latin + cyrillic) |

**Rationale:** This is a Telegram Mini App where bundle size directly affects load time inside the messenger WebView. Custom Tailwind components are lighter than any component library. Inter is the standard for modern product UI with excellent Cyrillic support. Lucide provides clean, consistent icons at minimal cost.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline element padding |
| sm | 8px | Compact element spacing, input internal padding, gap between slot grid cells |
| md | 16px | Card internal padding, between form fields, default element spacing, section content padding |
| lg | 24px | Between cards in a list, section-level vertical rhythm |
| xl | 32px | Page-level top/bottom padding |
| 2xl | 48px | Major section breaks (e.g., between booking flow steps) |

Exceptions:
- Touch targets: minimum 44px height for all tappable elements (buttons, time slot cells, list items). This is iOS HIG minimum and critical for thumb-friendly booking flow.
- Slot grid cells: 48px height to ensure comfortable touch targeting with time text.

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption | 12px | 400 (regular) | 1.33 (16px) | Secondary metadata, timestamps, helper text |
| Body | 14px | 400 (regular) | 1.43 (20px) | Default body text, descriptions, form labels |
| Title | 16px | 600 (semibold) | 1.25 (20px) | Card titles, section headers, nav items |
| Heading | 20px | 600 (semibold) | 1.2 (24px) | Page titles, step headers in booking flow |

Font stack: `'Inter', system-ui, -apple-system, sans-serif`

Notes:
- Mini-app viewport is constrained (typically 360-428px wide). 14px body ensures readability on small screens.
- Only 2 weights loaded (400 + 600) to minimize font download size.
- Russian text tends to be 20-30% longer than English equivalents. 14px body with 1.43 line-height accommodates longer Cyrillic strings without overflow.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) | Page background, input backgrounds, modal backgrounds |
| Secondary (30%) | `#F4F5F7` (cool gray-50) | Section backgrounds, card surfaces, time slot grid background, dividers |
| Accent (10%) | `#6C5CE7` (electric violet) | See reserved-for list below |
| Destructive | `#EF4444` (red-500) | Cancel booking button, destructive confirmations, error states |
| Success | `#10B981` (emerald-500) | Booking confirmed indicator, available slot indicator, success toast |
| Text primary | `#1A1A2E` (near-black blue) | Headings, body text, primary labels |
| Text secondary | `#6B7280` (gray-500) | Captions, placeholders, secondary info |
| Text on accent | `#FFFFFF` (white) | Text on accent-colored buttons |
| Border | `#E5E7EB` (gray-200) | Input borders, card borders, dividers |

**Accent `#6C5CE7` reserved for:**
1. Primary CTA buttons ("Записаться", "Далее", "Сохранить настройки")
2. Active booking flow step indicator (current step dot/bar)
3. Selected time slot highlight border + background tint (`#6C5CE7` at 8% opacity = `#6C5CE714`)
4. Selected date highlight in calendar
5. Bottom navigation active tab icon (master panel)
6. Loading spinner

**Accent is NOT used for:** links in body text (use text-primary with underline), icons in lists (use text-secondary), borders of inactive elements.

**Dark mode:** Not implemented in Phase 2. The mini-app uses `@media (prefers-color-scheme: light)` exclusively. Telegram's native dark mode does NOT affect mini-app content. Revisit in Phase 5 if needed.

## Component Inventory

### Client Booking Flow (5-step wizard)

| Step | Screen | Key Components |
|------|--------|----------------|
| 1 | Service Selection | `ServiceCard` (emoji + name + duration + price), scrollable list |
| 2 | Date Selection | `Calendar` (react-day-picker, ru locale, disabled past dates + days off) |
| 3 | Time Selection | `SlotGrid` (grid of 48px touch targets, columns of 2-3, available/unavailable/selected states) |
| 4 | Client Info | `BookingForm` (name input + phone input with +7 mask, auto-filled if returning client) |
| 5 | Confirmation | `BookingSummary` (service, date, time, price -- all in one card) + CTA button |

**Focal point (Step 1 -- Service Selection):** The service card list is the primary visual anchor. The first visible `ServiceCard` occupies the upper third of the content area (below heading + step indicator). Each card uses md (16px) internal padding, emoji at 32px size left-aligned, service name as Title (16px semibold), and duration + price as Body (14px regular, text-secondary) on a second line. This ensures the user's eye lands on the service options immediately upon entry.

**Flow navigation:**
- Linear wizard with back button (uses Telegram back button via platform bridge)
- Step indicator: horizontal dot/bar progress at top (5 dots, current = accent, completed = accent filled, upcoming = gray)
- No skip allowed. Steps must be completed in order.

### Master Management Panel

| Screen | Key Components |
|--------|----------------|
| Dashboard | `BookingCard` (time + client name + service, status badge), today's schedule list |
| Services | `ServiceCard` (CRUD), add button, swipe-to-delete or trash icon |
| Schedule | Weekly grid (Mon-Sun rows, start/end time pickers), break time toggle, buffer/deadline dropdowns |
| Bookings | Filterable list (date range, status), `BookingCard` with action buttons |
| Clients | Client list (name, phone, visit count), tap to view history |
| Settings | Profile form, buffer time selector, cancellation deadline selector |

**Master navigation:**
- Bottom tab bar: 5 tabs (Dashboard, Bookings, Services, Clients, Settings)
- Tab icons from Lucide: `calendar-days`, `book-open`, `scissors` (or relevant service icon), `users`, `settings`
- Active tab: accent color icon + label. Inactive: text-secondary icon, no label.
- Inactive tab icons must include `aria-label` with the tab name (e.g., `aria-label="Записи"` for `book-open`, `aria-label="Услуги"` for `scissors`, `aria-label="Клиенты"` for `users`, `aria-label="Настройки"` for `settings`, `aria-label="Сегодня"` for `calendar-days`). Active tabs already have visible labels and do not need `aria-label`.

### Shared Components

| Component | Spec |
|-----------|------|
| `Button` | Primary (accent bg, white text, h-44px, rounded-12px, font-semibold 14px). Secondary (white bg, accent text, accent border, same dimensions). Disabled (opacity-50, pointer-events-none). Full-width on mobile. |
| `Input` | h-44px, rounded-10px, border gray-200, focus ring accent at 30% opacity, 14px text, 12px placeholder. Error state: border red-500, error text 12px red-500 below. |
| `Card` | bg white, rounded-14px, border gray-200, padding 16px. Shadow: `0 1px 3px rgba(0,0,0,0.04)`. |
| `Badge` | Rounded-full, px-8px py-2px, 12px semibold text. Variants: confirmed (emerald-50 bg, emerald-700 text), pending (amber-50 bg, amber-700 text), cancelled (red-50 bg, red-700 text). |
| `Toast` | Fixed bottom, 16px from edges, rounded-12px, 14px text, auto-dismiss 3s. Success (emerald bg), error (red bg), info (gray-800 bg). White text. |
| `Skeleton` | Pulsing gray-200 rounded rectangles matching component dimensions. Used during API loading states. |
| `EmptyState` | Centered icon (48px, text-secondary) + heading (16px semibold) + body (14px text-secondary) + optional CTA button. |

## Interaction Contracts

### Booking Flow Interactions

| Interaction | Behavior |
|-------------|----------|
| Service tap | Card gets accent border, checkmark appears. Haptic feedback (light). Auto-advance to step 2 after 300ms delay. |
| Calendar date tap | Date cell fills with accent bg, white text. Haptic feedback (light). Auto-advance to step 3. Disabled dates (past, days off) are grayed out and non-tappable. |
| Time slot tap | Slot gets accent border + 8% accent bg tint. Haptic feedback (light). Previous selection deselected. "Далее" button appears/enables at bottom. |
| Form submission | "Записаться" button shows inline spinner. Disable button during API call. On success: navigate to confirmation. On error: shake animation on button, toast with error message. |
| Booking confirmation | Checkmark animation (scale-in 0 to 1 over 400ms). Haptic feedback (medium). "Мои записи" link below. |
| Slot unavailable (race) | If selected slot becomes unavailable between selection and submission: toast "Это время уже занято", redirect back to step 3 with refreshed slots. |
| Pull-to-refresh | Not used (mini-app constraint). Use manual refresh button in header for master panel lists. |

### Master Panel Interactions

| Interaction | Behavior |
|-------------|----------|
| Cancel booking (master) | Tap cancel icon -> confirmation bottom sheet -> confirm button (destructive red) -> API call -> success toast. |
| Add service | Tap "+" button -> full-screen form slides up from bottom -> fill fields -> "Сохранить настройки" -> list refreshes. |
| Delete service | Swipe left reveals red delete zone, or tap trash icon -> confirmation dialog. |
| Tab switch | Instant, no transition animation. Active tab indicator moves to new position. |

### Loading & Error States

| State | Behavior |
|-------|----------|
| Initial load | Full-screen skeleton matching page layout. 3 skeleton cards for lists, skeleton calendar for date picker. |
| Inline loading | Spinner inside button for form submissions. Skeleton cards appended for pagination. |
| API error (network) | Toast: "Нет соединения. Проверьте интернет и попробуйте снова." Retry button in empty state. |
| API error (server) | Toast: "Что-то пошло не так. Попробуйте позже." No retry auto-attempt. |
| Empty list | EmptyState component with contextual icon + copy (see Copywriting Contract). |

## Layout Contracts

### Mini-App Viewport

- Width: 100vw (fills Telegram WebView, typically 360-428px)
- Height: 100dvh (dynamic viewport height, accounts for Telegram header)
- No horizontal scroll. Vertical scroll per-section only.
- Safe area insets: respect `env(safe-area-inset-top)` and `env(safe-area-inset-bottom)` for notch/gesture-bar devices.
- Fixed bottom bar (CTA buttons in booking flow, tab bar in master panel) with 16px bottom padding + safe area.

### Booking Flow Layout

```
+----------------------------------+
| [<- back]  Step indicator dots   |
|  (Telegram back button)         |
+----------------------------------+
|                                  |
|  Heading (20px semibold)         |
|                                  |
|  Content area (scrollable)       |
|  - Service list                  |
|  - Calendar                     |
|  - Time grid                    |
|  - Form                        |
|                                  |
+----------------------------------+
|  [  Primary CTA Button  ]       |
|  (fixed bottom, 16px padding)   |
+----------------------------------+
```

### Master Panel Layout

```
+----------------------------------+
|  Heading (20px semibold)  [+]   |
+----------------------------------+
|                                  |
|  Content area (scrollable)       |
|  - Cards / List items           |
|                                  |
+----------------------------------+
|  [Tab] [Tab] [Tab] [Tab] [Tab]  |
|  (fixed bottom tab bar)         |
+----------------------------------+
```

### Slot Grid Layout

- 3 columns on screens >= 375px, 2 columns on smaller screens
- Each slot cell: 48px height, rounded-10px, centered time text (14px semibold)
- States: available (white bg, gray-200 border), selected (accent-8% bg, accent border, accent text), unavailable (gray-100 bg, gray-300 text, non-tappable)
- Gap between cells: 8px

### Calendar Customization (react-day-picker)

- Locale: `ru` (via dayjs locale)
- First day of week: Monday
- Disabled: past dates, dates where `is_working=false` or schedule exception `is_day_off=true`
- Selected date: accent bg, white text, rounded-full
- Today: accent text, no bg fill (dotted underline)
- Navigation arrows: Lucide `chevron-left` / `chevron-right`, 24px, text-secondary

## Accessibility Notes

- All touch targets >= 44px (iOS HIG minimum)
- Focus ring: 2px accent at 30% opacity, offset 2px (visible for keyboard nav in dev/web mode)
- Color contrast: text-primary on white = 14.5:1 (AAA), text-secondary on white = 4.6:1 (AA), white on accent = 4.7:1 (AA)
- Form inputs: associated labels, error messages linked via `aria-describedby`
- Calendar: keyboard navigable (react-day-picker built-in a11y)
- Inactive tab icons: must include `aria-label` attribute with the Russian tab name (see Master navigation section for exact values)
- Russian language throughout: no mixed-language strings in UI
