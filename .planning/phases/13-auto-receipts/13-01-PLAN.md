<objective>
Backend infrastructure for auto-receipts via Robokassa ReceiptAttach: INN binding API, ReceiptAttach HTTP service, PaymentService integration for cash/card payments, background retry with APScheduler, receipt link notification to client, error notification to master on failure.

Purpose: Enable self-employed masters to automatically issue tax receipts for any payment method (cash, card, SBP) through their existing Robokassa connection.
Output: ReceiptService, INN settings endpoints, payment flow integration, retry scheduler job.

1. **RobokassaService (extend)** -- add `build_receipt_attach_payload` static method:
   - Robokassa ReceiptAttach endpoint: POST `https://ws.roboxchange.com/RoboFiscal/Receipt/Attach`
   - Payload JSON:
     ```python
     {
       "MerchantLogin": creds.merchant_login,
       "Receipt": {
         "sno": master.receipt_sno,  # e.g. "patent"
         "items": [{
           "name": service_name[:128],
           "quantity": 1,
           "sum": float(amount_rub),
           "payment_method": "full_payment",
           "payment_object": "service",
           "tax": "none"
         }]
       },
       "Signature": computed_signature,
       "Type": "Income",
       "CustomerPhone": client_phone,  # E.164 format
       "Total": float(amount_rub)
     }
     ```
   - Signature for ReceiptAttach: hash(MerchantLogin:Total:Receipt_JSON:Password#1) using the master's configured hash algorithm
   - Receipt_JSON is the JSON-serialized Receipt dict (compact, no spaces, ensure_ascii=False), then URL-encoded
   - Return the full payload dict ready for HTTP POST

2. **ReceiptService (new file)** -- `backend/app/services/receipt_service.py`:
   - Class `ReceiptService` with static methods
   - `send_receipt_attach(db, payment_id)`:
     - Load payment with booking->service, booking->client relations
     - Load master

     - Build payload via `RobokassaService.build_receipt_attach_payload()`
     - Make HTTP POST to `https://ws.roboxchange.com/RoboFiscal/Receipt/Attach` using `httpx.AsyncClient` (httpx already in project deps)
     - If test mode: use `https://ws.roboxchange.com/RoboFiscal/Receipt/Attach` same URL but set IsTest=1 in payload
     - On success (HTTP 200 with ResultCode=0): update `payment.receipt_status = "issued"`, parse response for receipt URL and save to `payment.fns_receipt_url`, return True
     - On failure: increment `payment.fns_receipt_attempts`, if attempts >= 3 set `receipt_status = "failed"`, return False
     - Flush after update
   - `get_client_platform_for_notification(db, payment)` -- helper to find client's messenger platform (same pattern as payments.py line 107-116: look up ClientPlatform for booking.client_id, priority telegram > max > vk)
