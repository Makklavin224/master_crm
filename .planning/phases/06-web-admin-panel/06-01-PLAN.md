    **Backend additions (4 new endpoints + 1 model + 1 migration + bot handlers):**

    1. Create `backend/app/models/qr_session.py` -- QrSession model:
       - Fields: id (UUID PK), session_type (VARCHAR: 'qr' or 'magic_link'), master_id (UUID FK nullable -- set on confirm), token (VARCHAR unique -- the short-lived session token), status (VARCHAR: 'pending', 'confirmed', 'expired', 'used'), created_at (datetime, default utcnow), expires_at (datetime), access_token (VARCHAR nullable -- JWT set on confirm).
       - No master_id scoping needed (these are pre-auth sessions).

    2. Create Alembic migration for qr_sessions table. Run: `cd backend && uv run alembic revision --autogenerate -m "add_qr_sessions_table"` then `uv run alembic upgrade head`.

    3. Update `backend/app/models/__init__.py` to import QrSession.

    4. Add schemas to `backend/app/schemas/auth.py`:
       - `QrInitResponse(BaseModel)`: session_id (str), qr_payload (str).
       - `QrStatusResponse(BaseModel)`: status (str), access_token (str | None = None).
       - `QrConfirmRequest(BaseModel)`: session_id (str), tg_user_id (str).
       - `MagicLinkVerifyRequest(BaseModel)`: token (str).

    5. Add 4 endpoints to `backend/app/api/v1/auth.py`:
       - `POST /auth/qr/init` -- creates QrSession (type='qr', status='pending', expires in 5 min), generates UUID session_id, qr_payload is `https://t.me/{bot_username}?start=qr_{session_id}`. Returns QrInitResponse. No auth required.
       - `GET /auth/qr/status/{session_id}` -- looks up QrSession by id. If expired (now > expires_at) -> return {status: 'expired'}. If confirmed -> return {status: 'confirmed', access_token: session.access_token}. If pending -> return {status: 'pending'}. No auth required.
       - `POST /auth/qr/confirm` -- receives QrConfirmRequest. Looks up QrSession by session_id. Verifies status='pending' and not expired. Looks up Master by tg_user_id. Creates JWT. Sets session.access_token = jwt, session.status = 'confirmed', session.master_id = master.id. Saves. Returns 200 OK. This endpoint is called by the bot, not the web panel. No auth required (bot calls it internally).
       - `POST /auth/magic/verify` -- receives MagicLinkVerifyRequest. Looks up QrSession by token where type='magic_link'. Validates not expired, status='pending'. Looks up master via session.master_id. Creates JWT. Marks session as 'used'. Returns TokenResponse. No auth required.

    6. Update `backend/app/bot/handlers.py`:
       - Add handler for `/start qr_{session_id}` deep link: when user sends /start with qr_ prefix, extract session_id, call POST /auth/qr/confirm internally (direct DB call, not HTTP) with the tg_user_id from the message sender. Reply "Login confirmed" to the user.
       - Add handler for `/login` command: create QrSession (type='magic_link', status='pending', expires in 10 min), set master_id from the sender's tg_user_id lookup. Generate a unique token (UUID). Build URL: `{FRONTEND_URL}/admin/auth/magic?token={token}`. Send inline keyboard button with text "Login to admin panel" and url=built_url. If master not found by tg_user_id, reply "Account not found".
