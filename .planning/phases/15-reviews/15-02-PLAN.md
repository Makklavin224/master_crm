    1. Extend backend/app/schemas/review.py with new schemas:

       ```python
       class ReviewAdminRead(BaseModel):
           """Review as shown in master's admin panel (includes all statuses)."""
           id: uuid.UUID
           booking_id: uuid.UUID | None
           rating: int
           text: str | None = None
           client_name: str
           client_phone: str
           service_name: str
           master_reply: str | None = None
           master_replied_at: datetime | None = None
           status: str  # published, pending_reply, request_sent
           created_at: datetime
           model_config = {"from_attributes": True}

       class ReviewReplyRequest(BaseModel):
           """Master's reply to a review."""
           reply_text: str = Field(min_length=1, max_length=500)

       class ReviewsListResponse(BaseModel):
           reviews: list[ReviewAdminRead]
           total: int
       ```

    2. Create backend/app/api/v1/reviews.py with two endpoints:

       a. GET /reviews -- list master's reviews (master-authenticated)
          - Query params: status (optional filter: "all", "published", "pending_reply"), page (default 1), page_size (default 20)
          - Query: SELECT reviews WHERE master_id = current_master.id AND status != 'request_sent' (hide sentinel rows from admin view)
          - If status filter provided and not "all": add WHERE status = {status}
          - JOIN client (for name, phone) and booking -> service (for service_name)
          - Order by created_at DESC
          - Return ReviewsListResponse with total count and paginated list
          - Use selectinload for client and booking->service relationships

       b. PUT /reviews/{review_id}/reply -- master replies to a review
          - Path param: review_id (UUID)
          - Body: ReviewReplyRequest
          - Verify review.master_id == current_master.id (security: master can only reply to own reviews)
          - Verify review has rating > 0 (not a request_sent sentinel)
          - Update: review.master_reply = body.reply_text, review.master_replied_at = datetime.now(tz=UTC), review.status = "published" (replying publishes the review)
          - Return the updated ReviewAdminRead

    3. Register in backend/app/api/v1/router.py:
       ```python
       from app.api.v1.reviews import router as reviews_router
       api_v1_router.include_router(reviews_router, prefix="/reviews", tags=["reviews"])
       ```
